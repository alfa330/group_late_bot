# Информационная система мониторинга посещаемости Group Late Bot

Информационная система **Group Late Bot** представляет собой автоматизированный инструмент для мониторинга трудовой дисциплины и учета рабочего времени сотрудников. Система интегрирована с API платформы учета времени **Workpace** и функционирует непосредственно в мессенджере Telegram.

Данное руководство содержит описание ключевых функций системы, структуры формируемых отчетов и правил использования доступных команд для руководителей подразделений и ответственных сотрудников.

---

## Основные функциональные возможности системы

### 1. Оперативный мониторинг нарушений трудовой дисциплины
Система осуществляет непрерывный автоматический аудит активности персонала каждые 2 минуты. При обнаружении отклонений от установленного графика работы в рабочий чат Telegram направляются соответствующие уведомления:

*   **Опоздание:** Фиксируется в случае, если первая отметка о приходе на рабочее место осуществлена позже времени начала плановой смены (с учетом заданного допустимого порога, установленного в настройках системы).
*   **Ранний уход:** Фиксируется при регистрации отметки о выходе ранее запланированного времени окончания смены.
*   **Отсутствие на рабочем месте:** Направляется в случае отсутствия отметки о приходе сотрудника в течение 10 минут после планового начала его смены.
*   **Отсутствие отметки об уходе:** Направляется в случае отсутствия регистрации отметки о выходе в течение 10 минут после планового окончания смены.
*   **Подозрительная отметка:** Фиксируется, если биометрическая отметка сотрудника помечена системой как недостоверная (например, подозрение на использование фотографии или необходимость ручной модерации). Уведомление содержит информацию об устройстве и точном времени регистрации.

### 2. Координация действий менеджеров (Кнопка «Отбито»)
Каждое оперативное уведомление оснащено кнопкой «Отбито» для взаимодействия руководителей непосредственно в рабочем пространстве Telegram:
*   Ответственный менеджер, обработавший инцидент (выяснивший причину отсутствия или опоздания сотрудника), нажимает кнопку «Отбито».
*   Сообщение мгновенно обновляется: в нем фиксируется ФИО менеджера, принявшего задачу, и точное время отметки.
*   Это исключает дублирование управленческих действий и координирует работу руководящего состава.

### 3. Персональное, глобальное и полное отключение отбивок
Пользователи могут временно отключать оперативные отбивки для конкретного сотрудника, целого отдела или полностью все уведомления в своем личном чате с ботом. Администратор может применять те же команды глобально для всех чатов рассылки или точечно для выбранного рабочего чата.

*   Отключение применяется к оперативным уведомлениям об опозданиях, ранних уходах, неявках и подозрительных биометрических отметках.
*   Фильтр работает по ФИО сотрудника и названию отдела, поэтому пользователь может отключить как одного человека, так и весь контур подразделения.
*   Команды `/mute_all` и `/unmute_all` позволяют временно выключить или вернуть все оперативные уведомления.
*   Персональные настройки привязаны к `chat_id`: если пользователь отключил отдел в личном чате, другие чаты продолжат получать эти уведомления.
*   В групповых чатах персонально скрыть сообщение от одного участника невозможно, поэтому обычные пользователи настраивают mute в личном чате с ботом.
*   Список отключений сохраняется в файле `mutes.json` и автоматически загружается после перезапуска сервиса.
*   Пользователь может в любой момент посмотреть свои отключения командой `/muted` и вернуть уведомления обратно командами `/unmute_user`, `/unmute_dept` или `/unmute_all`.

### 4. Маршрутизация уведомлений по отделам
Администратор может закрепить рабочий групповой чат за одним или несколькими отделами. После настройки такой чат получает только уведомления сотрудников выбранных отделов, а отчеты из этого чата по умолчанию строятся по тем же отделам.

*   Список отделов берется из официального эндпоинта Workpace `/public/v1/employee`.
*   Название отдела для отчетов и уведомлений определяется по карточке сотрудника, а не по названию графика.
*   Команда `/departments` показывает текущий список активных отделов для проверки написания.
*   Команды `/setchatdept [Chat ID] [Отдел1; Отдел2]` и `/setdept [Отдел1; Отдел2]` закрепляют чат за одним или несколькими отделами. Несколько отделов можно разделять символами `;` или `|`.
*   Команды `/addchatdept [Chat ID] [Отдел]`, `/adddept [Отдел]`, `/delchatdept [Chat ID] [Отдел]` и `/deldept [Отдел]` позволяют точечно менять список отделов без полной перезаписи.
*   Команды `/clearchatdept [Chat ID]` и `/cleardept` снимают фильтр отдела.

### 5. Автоматизированная отчетность в формате Excel
По запросу система генерирует структурированную аналитическую таблицу в формате Excel. Документ оформлен в деловом корпоративном стиле и включает следующие параметры:

1.  **Отдел** — закрепленное подразделение сотрудника.
2.  **ФИО сотрудника** — персональные данные сотрудника.
3.  **График** — наименование назначенного рабочего расписания.
4.  **Все отметки за день** — полная хронологическая последовательность всех приходов и уходов за текущие сутки. Подозрительные отметки помечаются специальным символом предупреждения.
5.  **Время прихода (план)** — плановое начало рабочего дня.
6.  **Время прихода (факт)** — фактически зафиксированное время первого прихода.
7.  **Опоздание (мин)** — количество минут опоздания относительно плана (при раннем приходе выводится прочерк).
8.  **Время ухода (план)** — плановое окончание рабочего дня.
9.  **Время ухода (факт)** — фактически зафиксированное время последнего выхода.
10. **Ранний уход (мин)** — количество минут отсутствия до планового окончания смены.
11. **Отработано (факт) (HH:MM)** — фактическая продолжительность нахождения сотрудника на рабочем месте (интервал между самым ранним входом и самым поздним выходом за день).
12. **Отклонение (HH:MM)** — разница между фактическим временем работы и установленной нормой смены (значение со знаком «+» означает переработку, со знаком «-» — недоработку).
13. **Статус** — итоговая оценка посещаемости с цветовой индикацией:
    *   **Вовремя** (светло-зеленый фон) — соответствие графику.
    *   **Опоздал / Ранний уход / Отсутствует** (светло-красный фон) — зафиксированы нарушения.
    *   **Вне графика** / **Вовремя (с предупреждением)** (светло-желтый фон) — работа в выходной день или наличие отметок, требующих подтверждения.

## Архитектура отчетов за период (Сводные отчеты)

При запросе аналитики за диапазон дат (период) система формирует многостраничную книгу Excel с разделением на следующие логические разделы:

*   **Лист 1: Сводная аналитика по сотрудникам (Дашборд)**
    Содержит консолидированные показатели эффективности за весь выбранный период для каждого сотрудника:
    *   Общее количество дней по графику (план выходов).
    *   Фактическое число отработанных дней.
    *   Количество дней, отработанных полностью без нарушений графика.
    *   Количество и суммарное время всех зафиксированных опозданий.
    *   Количество и суммарное время всех ранних уходов.
    *   Общее количество неявок (прогулов).
    *   Суммарно отработанное время за период.
    *   Среднее рабочее время за один рабочий день.

*   **Последующие листы (Детальные ежедневные отчеты)**
    Каждый день периода выносится на отдельную вкладку, названную соответствующей датой. Вкладка содержит подробную суточную таблицу со всеми отметками и расчетами, отфильтрованную по выбранным параметрам.

## Оптимизация и защита данных

В систему встроены интеллектуальные модули для повышения точности учета и стабильности работы:
*   **Коррекция ранних приходов (Алгоритм самовосстановления):** Если сотрудник отметился слишком рано или отметка ожидает подтверждения, стандартный алгоритм API Workpace может не связать её со сменой. Бот автоматически анализирует первичные аппаратные логи отметок на входе и восстанавливает реальное время прихода, предотвращая ложные уведомления об отсутствии.
*   **Исключение архивных записей:** Уволенные и временно отстраненные сотрудники автоматически исключаются из всех видов отчетов и рассылок.
*   **Гибкое отключение отбивок:** Пользователи могут персонально отключать уведомления по отдельным сотрудникам или отделам, а администратор может применять такие исключения глобально без изменения расписаний и отчетной логики.
*   **Контроль повторных запросов:** Внедрена система предотвращения генерации дубликатов отчетов при множественных нажатиях команд пользователями в Telegram.

## Список команд системы в Telegram

*   `/start` — Инициализация бота, отображение идентификатора текущего чата (Chat ID) и краткого перечня функций.
*   `/help` — Просмотр подробного регламента и руководства по использованию системы.
*   `/report` — Генерация Excel-отчета за **текущие сутки** по всем подразделениям компании.
*   `/report [Название Отдела]` — Генерация отчета за **текущие сутки** с фильтрацией по конкретному отделу (например, `/report Контакт-центр`).
*   `/report ГГГГ-ММ-ДД` — Генерация отчета за **выбранную дату** по всем отделам (например, `/report 2026-05-18`).
*   `/report ГГГГ-ММ-ДД [Название Отдела]` — Генерация отчета за выбранную дату по конкретному отделу (например, `/report 2026-05-18 Контакт-центр`).
*   `/report ГГГГ-ММ-ДД ГГГГ-ММ-ДД` — Генерация **сводного отчета за указанный период** по всем отделам (например, `/report 2026-05-18 2026-05-20`).
*   `/report ГГГГ-ММ-ДД ГГГГ-ММ-ДД [Название Отдела]` — Генерация **сводного отчета за период по конкретному отделу** (например, `/report 2026-05-18 2026-05-20 Контакт-центр`).
*   `/addchat [Chat ID]` — *(Доступно администраторам)* Подключение нового чата в контур автоматической рассылки уведомлений о нарушениях.
*   `/delchat [Chat ID]` — *(Доступно администраторам)* Отключение чата от контура автоматической рассылки.
*   `/chats` — *(Доступно администраторам)* Отображение перечня всех подключенных рабочих чатов.
*   `/departments` — Просмотр списка активных отделов Workpace для проверки точного названия.
*   `/setchatdept [Chat ID] [Отдел1; Отдел2]` — *(Доступно администраторам)* Закрепление указанного чата за одним или несколькими отделами.
*   `/setdept [Отдел1; Отдел2]` — *(Доступно администраторам)* Закрепление текущего чата за одним или несколькими отделами.
*   `/addchatdept [Chat ID] [Название Отдела]` — *(Доступно администраторам)* Добавление отдела к уже закрепленному списку указанного чата.
*   `/adddept [Название Отдела]` — *(Доступно администраторам)* Добавление отдела к уже закрепленному списку текущего чата.
*   `/delchatdept [Chat ID] [Название Отдела]` — *(Доступно администраторам)* Удаление отдела из закрепленного списка указанного чата.
*   `/deldept [Название Отдела]` — *(Доступно администраторам)* Удаление отдела из закрепленного списка текущего чата.
*   `/clearchatdept [Chat ID]` — *(Доступно администраторам)* Снятие фильтра отдела с указанного чата.
*   `/cleardept` — *(Доступно администраторам)* Снятие фильтра отдела с текущего чата.
*   `/mute_user [ФИО]` — Временное отключение оперативных отбивок по конкретному сотруднику для текущего пользователя.
*   `/unmute_user [ФИО]` — Повторное включение оперативных отбивок по сотруднику для текущего пользователя.
*   `/mute_dept [Название Отдела]` — Временное отключение оперативных отбивок по всему отделу для текущего пользователя.
*   `/unmute_dept [Название Отдела]` — Повторное включение оперативных отбивок по отделу для текущего пользователя.
*   `/mute_all` — Временное отключение всех оперативных уведомлений. Для администратора без аргументов применяется глобально.
*   `/unmute_all` — Повторное включение всех оперативных уведомлений.
*   `/mute_all [Chat ID]` — *(Доступно администраторам)* Временное отключение всех уведомлений только для указанного чата.
*   `/unmute_all [Chat ID]` — *(Доступно администраторам)* Повторное включение всех уведомлений для указанного чата.
*   `/muted` — Просмотр списка сотрудников и отделов, для которых отбивки временно отключены.
