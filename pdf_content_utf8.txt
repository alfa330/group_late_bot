1.1. Тест и продакшн сервера
Тестовый сервер:
● GATEWAY https://api.integrations-testing.workpace.kz/
● FRONT https://my.integrations-testing.workpace.kz/
Продакшн сервер:
● GATEWAY https://api.workpace.kz/
● FRONT https://my.workpace.kz/
Примечание: Для получения доступа к запросам пользователю, под которым
происходит авторизация, должно быть выдано право “Интеграция с внешними
системами”.

=== PAGE 3 ===
1.2. Авторизация
POST api/auth/
Request body
{
"login": "string",
"password": "string"
}
Response body
{
"accessToken": "string",
"refreshToken": "string"
}
1.3. Обновление токена
POST api/auth/refresh
Request headers: Authorization = Bearer token
Request body
{
"accessToken": "string",
"refreshToken": "string"
}
Response body
{
"accessToken": "string",
"refreshToken": "string"
}
Токен доступа живёт 30 минут.
Рефреш токен живёт 23 часа. Если в течение 23 часов не было обновлений - токен
доступа отзовётся и его нужно будет запросить снова.

2.4. Получение списка сотрудников
GET /public/v1/employee
Request headers: Authorization = Bearer token
Поля в Query запроса
Параметр Обяз. Тип Комментарий
filter Нет string Параметр для фильтрации сотрудников.
Укажите в фильтре:
● IsArchived = true, если требуется
получить только Сотрудников в
статусе “Удален”
● IsArchived = false, если требуется
получить только Сотрудников в
статусе “Активен”
● оставьте фильтр по IsArchived
пустым если требуется получить
всех сотрудников
skip Нет integer Пропуск заданного числа элементов
take Нет integer Возвращение заданного числа элементов
requiretotalcount Нет boolean Укажите true, если требуется вернуть
значение totalCount (общее количество
элементов в системе)
Пример:
Запрос для получения только активных сотрудников:
/public/v1/employee?filter=%5B%22isArchived%22,%22=%22,false%5D&requireTotalCount=true
Запрос для получения всех сотрудников, включая удаленных:
/public/v1/employee?requireTotalCount=true
